Dig up the web !

Digger est un projet client permettant de traquer et centraliser les données publiques que nous pouvons laisser derrière nous sur le web. À l’aide de différents outils intégrés, Digger permet de retrouver des informations enfouies sur le web.

J’ai réalisé ce projet lors de mon semestre 4 à INTECH en tant que développeur et chef de projet.

Objectifs

Tout d’abord qu’est-ce que l’OSINT ? OSINT ou Open Source Intelligence (fouille de données en source ouverte) est une technique de renseignement utilisé dans les secteurs militaires et de l’intelligence économiques qui permet de retrouver des informations sur quelqu’un ou une entité (comme par exemple une entreprise) dans des bases de données ouvertes, comme Google, Facebook, Twitter, etc.

En sachant cela l’objectif de ce projet client était de créer une solution OSINT qui soit :

  • User-friendly
  • Développer une infrastructure permettant de décentraliser la recherche des données
  • Accessible n’importe où
  • Et les éléments pédagogiques suivants :
    • Architecture 3 tiers
    • Single Page Application
    • Sécurité

Résultats

Digger est l’un des projets où j’ai le plus progresser techniquement notamment sur les concepts de l’OSINT, gérer un cahier des charges imposés par un client extérieur.
J’ai crée un schéma d’architecture complexe composé de deux serveurs et de deux bases de données, sécuriser la communication entre les deux serveurs, créer des points d’API avec un système d’authentification par rôle pour les utilisateurs en C#, créer des scripts python pour réutiliser des applications sous Linux et les Dockeriser, etc.

La valeur ajoutée du projet était la possibilité d’ajouter soi-même des applications, par exemple si vous aviez un logiciel d’OSINT fonctionnelle, vous n’aviez qu’à dockeriser celui-ci et l’ajouter via un formulaire du site web pour qu’il soit intégré, de cette façon le client n’a pas besoin de nous pour améliorer le produit.

Analyse post-mortem

Ce projet était ambitieux, effectivement lors de la phase de développement, nous avions pris beaucoup de retard, car les technologies que nous devions utiliser nous étaient inconnues, de plus les différents logiciels d’OSINT qui devait être intégré à notre application n’était pas adaptés et j’ai dû modifier leurs codes sources à la main pour répondre aux exigences de notre application.

Cependant, je pense que cela a été intéressant dans le fait où j’ai dû redoubler de créativité pour surmonter les problèmes et c’est ce que j’apprécie dans le développement.
