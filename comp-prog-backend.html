Programmation backend

Le backend en programmation est la partie que je préfère dans mon métier, à travers ma formation et mon expérience professionnelle, j'ai acquis une bonne expertise principalement en TypeScript avec le framework NestJS, Java, Python et SQL où j'ai travaillé sur différents sujets comme le développement d'une plateforme de vente de produit ou encore la mise en place d'un service permettant le suivi des factures de comptabilité.

Typescript - JavaScript - NestJS

Durant mes premières années, j'ai développé plusieurs applications à l'aide du langage JavaScript, ce que je n'aime pas dans ce langage est son faible typage qui apporte son lot de problème notamment sur la documentation du code.

Vint mon stage chez Nodata et j'ai découvert le TypeScript qui est un langage libre et open source développé par Microsoft qui a pour but d'améliorer le JavaScript notamment sur son typage.
De cette découverte, je me suis mis à apprécier à code dans ce langage que j'ai mis en pratique sur du frontend, mais surtout sur du backend avec le framework NestJS lors de mon alternance chez Advanced schema.

Mes réalisations sur cette compétence :

  • Nodata
  • Advanced Schema

Java

J'ai découvert le Java récemment lors de mon alternance chez Advanced Schema où j'ai crée un back-office.

Je n'ai pas eu de mal à apprendre le langage du fait de ma pratique du C# qui est très similaire au Java en terme de concept.

Ma réalisation sur cette compétence :

  • Advanced Schema

Python

Je code en Python depuis maintenant plusieurs années, la facilité d'écriture du code et qu'il soit portable sur tous les systèmes, faits que mon langage de prédilection pour écrire des scripts système ou tester un concept rapidement.

Mes réalisations sur cette compétence :

  • Digger

  • Advanced Schema

Base de données

Le langage SQL qui permet d'interagir avec une bonne de donnée et le langage avec lequel j'ai été le plus régulier, je ne crois pas avoir passé un mois sans l'utiliser notamment lors de mes projets étudiants, mais aussi en entreprise où je l'ai mis en pratique pour corriger et ajouter des fonctionalités dans un datawhare (concept d'entrepôt de données).

J'ai travaillé sur les principaux types de base de données classique comme MySQL, MariaDB, PostgreSQL et SQL Server (Microsoft).
J'ai également travaillé sur des bases de données de NoSQL comme YugabyteDB, Redis et ElasticSearch lors de mon stage et mon alternance chez Nodata, mais également sur Dgraph qui est une base de données orientée graphe sur mon projet étudiant Digger.

Mes réalisations sur cette compétence :

  • Trump Tower
  • Digger
  • Nodata
  • Advanced Schema
